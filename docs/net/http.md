# HTTP

## HTTP 报文

## HTTP 常见状态码

状态码的职责是当客户端向服务器端发送请求时，描述返回的请求结果。借助状态码，用户可以知道服务器端是正常处理了请求，还是出现了错误。

| 状态码 | 英文名称     | 中文描述                                             |
| ------ | ------------ | ---------------------------------------------------- |
| 200    | ok           | 客户端的请求已被服务器正常处理                       |
| 204    | No Content   | 客户端的请求已被服务器正常处理，但没有返回内容       |
| 206    | Partial      | 客户端进行了范围请求，服务器成功执行了这部分(视频流) |
| 301    | Moved        | 永久性重定向                                         |
| 302    | Found        | 临时性重定向                                         |
| 303    | See Other    | 临时性重定向，表示重定向要以 get 方式请求            |
| 304    | Not          | 客户端发送附带条件的请求时，服务器端允许请求         |
| 307    | Temporary    | 临时性重定向                                         |
| 400    | Bad          | 服务器端无法理解客户端发送的请求                     |
| 401    | Unauthorized | 客户端请求需要用户认证                               |
| 403    | Forbidden    | 服务器端拒绝该次请求                                 |
| 404    | Not Found    | 客户端请求的资源不存在                               |
| 500    | Internal     | 服务器端在执行请求时发生了错误                       |
| 503    | Service      | 服务器暂时性的无法处理客户端的请求                   |
