# AI 辅助开发实践笔记

AI 更适合参与有明确验收条件的工程任务。模糊目标容易得到看似完整但无法验证的结果，明确命令和证据会让协作质量明显提升。

## 更有效的输入

- 给出目标范围，例如“只改 `docs/posts` 下的文章结构”。
- 给出硬性约束，例如“必须使用 Rspress，不引入额外前端框架”。
- 给出验收命令，例如“完成后运行 `pnpm run lint` 和 `pnpm run build`”。

## 协作节奏

先让 AI 阅读项目结构和现有约定，再要求它实现。完成后不要只看总结，要看实际验证结果。对于前端站点，还应该打开本地页面，检查导航、文字换行、主题色和移动端布局。

## 适合沉淀的内容

- 常见错误和对应修复步骤。
- 项目初始化清单。
- 发布流程和回滚方式。
- 组件或页面迁移时的测试用例。
