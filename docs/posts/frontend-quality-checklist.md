# 前端项目质量检查清单

一个前端项目能否长期维护，通常取决于日常流程是否稳定，而不是某一次大规模重构。下面是我会优先保留的检查项。

## 基础命令

- `pnpm run lint`: 发现语法、Hooks、未使用声明和基础风格问题。
- `pnpm run build`: 确认生产构建、路由和静态资源没有断裂。
- `pnpm run preview`: 本地预览生产构建，检查真实输出效果。

## 内容检查

- 导航链接是否能从首页进入。
- 每篇文章是否有清晰的标题和可扫描的小节。
- 代码块是否能独立复制执行，必要时补齐上下文。
- 图片、Logo、静态资源是否位于 `docs/public/` 并通过绝对路径引用。

## 发布检查

发布前确认构建产物目录是 `doc_build`。GitHub Pages workflow 应上传这个目录，而不是源代码目录或开发服务器输出。
