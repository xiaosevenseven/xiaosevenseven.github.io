# 模块化开发

模块化的开发方式可以提高代码复用率，方便进行代码的管理。通常一个文件就是一个模块，有自己的作用域，只向外暴露特定的变量和函数。

## CommonJS

Node.js 是 CommonJS 规范的主要践行者。这种模块加载方案是服务器端的解决方案，它是以同步的方式来引入模块的，因为在服务器端文件都是存储在本地磁盘，所以读取非常快，所以以同步的方式加载没有问题。但是在浏览器端，文件都是存储在服务器端，所以需要下载，如果采用同步加载，会造成浏览器失去响应，所以浏览器端一般采用异步加载。

```js
// 定义模块 a.js
const title = "前端";
function say(name, age) {
  console.log(`我是${name}，今年${age}岁，欢迎关注我～`);
}
// 在这里向外暴露函数和变量
module.exports = {
  title,
  say,
};
```

```js
// 定义模块 b.js
// 引入自定义模块
const a = require("./a.js");
a.say("Seven", 18);
```

## ESModule

ES6 在语言标准的层面上，实现了模块功能，而且实现的相当简单，旨在成为浏览器和服务器通用的模块解决方案。其模块功能主要由 export 和 import 组成。

- export 命令用于规定模块的对外接口，
- import 命令用于输入其他模块提供的功能。

```js
// 定义模块 a.js
const title = "前端";
function say(name, age) {
  console.log(`我是${name}，今年${age}岁，欢迎关注我～`);
}
// 在这里向外暴露函数和变量
export { title, say };
```

```js
// 定义模块 b.js
// 引入自定义模块
import { title, say } from "./a.js";
say("Seven", 18);
```

## 差异

1. CommonJS 模块输入的是一个值的拷贝，ES6 模块输入的是值的引用。

- CommonJS 模块输入的是值的拷贝，也就是说，一旦输出这个值，模块内部的变化不会影响到这个值。
- ES6 模块的运行机制与 CommenJS 不一样。JS 引擎对脚本静态分析的时候，遇到模块加载命令 import，就会去生成一个只读引用。等脚本真正执行的时候，再根据这个只读引用，到被加载的模块里去取值。

2. CommonJS 模块是运行时加载，ES6 模块时编译时输入接口。

- 运行时加载：CommonJS 模块就是对象，即在输入时是先加载整个模块，生成一个对象，然后再从这个对象上读取方法，这种加载叫运行时加载。
- 编译时加载：ES6 模块不是对象，而是通过 export 命令显式指定输出的代码，import 时采用静态命令的形式，即在 import 时可以指定加载某个输出值，而不是加载整个模块，这种加载称为编译时加载。
